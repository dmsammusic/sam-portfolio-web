My Journey to Building a Personal Website

As an engineer and a lifelong learner, I’ve always enjoyed solving problems and creating solutions. Recently, I decided to build my personal website to share journals and thoughts about changes in life and technology. Here’s how it all came together in two days during my free time.

Setting the Foundation

First, I bought a domain from GoDaddy. That was simple, but I knew there was more work ahead. To make my site secure with HTTPS, I used Cloudflare for SSL and DNS setup. After some quick configuration, my domain was ready, and I could start building the website.

Designing the Website

I wanted a space to post journals and blogs, so I decided to use AI-powered tools to speed up the process.

Choosing the Tools

To start with the design, I used MagicPatterns, which gave me a basic design using Tailwind CSS and a React page. It provided a good foundation, but I wanted the design to feel more personal.

Customizing the Design

With my basic UI skills and help from ChatGPT, I adjusted the design to match my style. This combination helped me finish the page design quickly.

Hosting the Website

Once the design was ready, I needed a web server to host the site. At first, I thought about using AWS S3 for hosting a static site. But after about an hour of research, I found GitHub Pages. It seemed perfect for my needs because it automatically updates the site whenever I push code changes.

Setting Up the Hosting

I uploaded a simple index page to GitHub Pages. The setup was easy, and I loved how it updated automatically whenever I pushed changes to the repository.

Adding the Blog Feature

With the hosting set up, I decided to add a blog page. Once again, I worked with ChatGPT to fine-tune the design and functionality. After testing the site on mobile devices, I realized it needed some adjustments.

Refining Mobile Views

To make sure the site worked well on mobile devices, I manually adjusted the design. The Tailwind CSS documentation helped me learn and apply responsive design techniques. This hands-on experience improved my skills and gave me better control over the final result.

Reflections on the Journey

The whole process—from buying the domain to creating the blog—took just two days during my free time. Along the way, I learned a few key lessons:

  • Focus on Solving Problems: As an engineer, I’ve learned to focus on solving problems rather than just chasing new technology.
  • Programming is a Tool: It’s about communicating with machines to make them work for us. AI tools like ChatGPT became a helpful part of my toolkit.
  • AI is a Helper: Like calculators, AI helps us focus on higher-level problems, enabling innovation and creativity.

Final Thoughts

Building this website wasn’t just a technical project—it reflected my approach to engineering and problem-solving. Humans are great at making decisions and using tools to achieve their goals. In this modern AI era, we have the chance to use these tools to innovate and create better things.

I’m excited to keep learning and sharing my experiences. Here’s to solving problems, thinking big, and making the most of the tools we have!
